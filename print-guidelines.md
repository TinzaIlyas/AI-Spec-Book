# Print-Ready Guidelines for Physical AI & Humanoid Robotics Book

## Physical Specifications

### Page Size
- **Standard**: A4 (210mm × 297mm)
- **Alternative**: US Letter (8.5" × 11")
- **For international compatibility**: A4 recommended

### Margins
- **Top/Bottom**: 1 inch (2.54cm)
- **Inside (binding side)**: 1.2 inch (3.05cm)
- **Outside**: 0.8 inch (2.03cm)
- This allows for proper binding and readability

### Typography

#### Main Text
- **Font**: Times New Roman (for Roman Urdu compatibility)
- **Size**: 11-12pt for body text
- **Line spacing**: 1.5x or 18pt
- **Justification**: Justified for clean edges

#### Headings
- **Chapter titles**: 18-20pt, bold
- **Section headings**: 14-16pt, bold
- **Subsection headings**: 12-14pt, bold

#### Code Blocks
- **Font**: Consolas, Courier New, or Liberation Mono
- **Size**: 10pt (slightly smaller than body text)
- **Background**: Light gray (10% shade)
- **Line spacing**: Single space for compactness

### Colors
- **Text**: 100% black (CMYK: C:0 M:0 Y:0 K:100)
- **Code blocks**: Dark gray text (K:80) on light gray background (K:10)
- **Links**: Avoid color in print version; use underlines if needed

## Content Structure

### Chapter Organization
- Each chapter should start on a right-hand (odd-numbered) page
- Include proper spacing between sections
- Use consistent heading hierarchy

### Table of Contents
- Include up to 2-3 heading levels
- Page numbers aligned with dots or tabs
- Ensure all chapters are properly listed

### Page Numbering
- Roman numerals (i, ii, iii) for front matter
- Arabic numerals (1, 2, 3) for main content
- Page numbers at bottom center or outer corner

## Code Block Handling

### For Print
- Reduce font size slightly to fit longer lines
- Wrap long lines appropriately
- Use consistent syntax highlighting colors that work in grayscale
- Consider using different background shades instead of colors for syntax elements

### Listings
- Number important code blocks as figures
- Add captions explaining the code's purpose
- Keep line numbers for reference purposes

## Images and Figures

### Resolution
- Minimum 300 DPI for all images
- Vector graphics preferred for diagrams
- Ensure all text in images is readable at print size

### Placement
- Position near relevant text
- Add proper captions
- Include "Figure X.Y" labels

## Quality Checks

### Before Final Print
- [ ] Verify all Roman Urdu text displays correctly
- [ ] Check that code blocks are legible
- [ ] Confirm page formatting is consistent
- [ ] Verify table of contents links are correct
- [ ] Ensure no text runs off page edges
- [ ] Check for proper hyphenation and line breaks
- [ ] Verify all images are high resolution

### Test Print
- Print sample pages to check formatting
- Verify readability of body text
- Check that margins are adequate
- Ensure headings stand out appropriately

## Special Considerations for Roman Urdu

### Text Direction
- Roman Urdu is written left-to-right like English
- Ensure proper font support for Urdu characters
- Check that mixed English-Urdu text displays correctly

### Typography
- Use fonts that properly support Romanized Urdu
- Maintain consistent transliteration style
- Consider using italics sparingly in body text

### Layout
- Ensure proper spacing around mixed scripts
- Check that code examples with Roman Urdu comments render correctly
- Verify that technical terms maintain clarity

## File Preparation

### PDF Settings
- Use PDF/X-1a standard for print compatibility
- Embed all fonts
- Set color profile to CMYK
- Include crop marks if required by printer

### File Naming Convention
- Version: Physical_AI_Humanoid_Robotics_v1.0_Print.pdf
- Include version number and "Print" indicator
- Use underscores instead of spaces

Following these guidelines will ensure your "Physical AI & Humanoid Robotics" book is properly formatted for professional printing.
